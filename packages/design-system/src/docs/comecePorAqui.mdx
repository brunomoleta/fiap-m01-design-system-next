### Tá, mas antes de começar... por que se cria um Design System?

Boa pergunta. Este espaço é utilizado para ser uma _praça pública_
entre equipes de Design, Desenvolvimento e Qualidade.

Como o único time que tem acesso ao código fonte é o de Desenvolvimento,
este canal irá facilitar o alinhamento de todas as pessoas
que trabalham na empresaX, garantindo o afinamento entre produto projetado e executado.

### Utilidade para a equipe de desenvolvimento

Você já usou o [Material UI](https://m3.material.io/)? Ele é a biblioteca de componentes da Google.

Este DS funciona com o mesmo intuito, é a biblioteca de componentes da empresaX.
Aqui é possível produzir e testar componentes de maneira independente do código,
um _laboratório front end_, por assim dizer.

---
#### Benefícios como um todo

- **Consistência**: Nosso produto terá aparência e comportamento consistente, criando uma experiência estável para a usuário.
- **Eficiência**: Componentes reutilizáveis significam menos trabalho repetitivo, e mais tempo para aspectos criativos e inovadores.
- **Manutenção Simplificada**: Atualizações em todo o sistema podem ser feitas centralmente, garantindo o produto atualizado integralmente.
- **Escalabilidade**: À medida que criamos novas funcionalidade, podemos alavancar o Design System para uma expansão mais suave.
